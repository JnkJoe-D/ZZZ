## 工作概览
今日的开发工作经历了深刻的架构洗盘。我们的核心目标非常清晰——**万物归一，消除特例**。
曾几何时，“移动动作 (`Jog` 等)”走的是传统 `AnimController.PlayAnim` 硬编码，而“战斗/闪避”走的则是 `SkillManager` 的时间轴，这导致如果希望在平跑时加上特定关键帧的沙尘特效或音效，将极为繁杂痛苦。
如今，通过抽离全新的 `ActionConfigSO` 基类和 `ActionPlayer` 动作枢纽，真正意义上实现了**所有角色动效均由时间轴统筹调度的蓝图**。在此基础上，极大程度地简化了原本臃肿的 FSM 状态机逻辑，使各大 State 完全回归其判断与流转的本质。
另外在迭代测试尾声时，修复了多个隐蔽且致命的问题，如底层资产异步序列化类型的歧义误判、以及长时间挂机后反复调用积累出的订阅事件风暴危机。

随后，为了进一步完善战斗核心体验并限制无敌帧的滥用，我们针对闪避系统补充开发了“基于独立充能的受限冷却机制”。现在闪避会有统一配置的连闪次数与充能周期，使得战斗的资源管控更为立体。同时还随手解决了技能播放完毕后因缺乏自动回流通道导致角色“卡死”在战斗态的细微收尾缺陷。

## 详细工作记录

### 1. 架构梳理与核心部件剥离 (Action System)
*   **基础配置分离 (`ActionConfigSO`/`LocomotionConfigSO`)**：结束了战斗技能包揽一切的历史，分离出专门承载移动参数扩展的数据底座。
*   **`ActionPlayer` 集成**：它是 `CharacterEntity` 的直属贴身幕僚，完全隐藏了繁杂的 Timeline 寻址、校准转向、获取 `ProcessContext` 和缓存重播细节。调用方只要执行极简的三板斧（`PlayAction`/`Tick`/`StopAction`）即可，真正实现了解耦。
*   **`ActionManager` 底层泛化**：针对热装载管线进行参数接口扩大，如今连“原地待机”的数据也会经过这里以内存池缓存姿态进驻游戏实战。

### 2. 状态机 (FSM) 逻辑大幅度瘦身与净化
*   **基础移动微状态 (`GroundSubState`)**：全盘清扫了 `GroundIdle`、`GroundJog`、`GroundDash` 和 `GroundStop` 中通过传统 API 驱动动作的包袱。
*   **动作响应即时抢占**：通过直接对 `ActionPlayer` 塞入主角 `Config` 中的最新配置引用（由原硬编码的动画名升级为 Inspector 上的 Timeline），甚至不需要再书写复杂的淡入淡出参数代码，全都交由全局系统保障。
*   **战斗与闪避态 (`CharacterSkillState/EvadeState`)**：原先两百多行的恶臭 `Runner` 分配模版代码被全部擦除。只留下最纯粹的 `InputAvailable` 参数提取和输入指令缓冲释放，大幅增加了日后介入修改和阅读的友好度。

### 3. 深耕引擎底层的顽症治愈 (Bug Fixes)
*   **AudioClip 伪“子资产”装载崩溃**：由于昨天的系统为 `SkillAudioClip` 进行了多变体扩建并附魔了 `AssetPath` 序列化信息。由于底层 `SerializationUtility` 的正则判读过于宽泛（只要有名字有路径不是 GameObject 就去 `LoadSubAsset`），造成 `.wav` 单文件载入引擎时爆红。现已精准约束只针对 `AnimationClip` 及 `AvatarMask` 分解，问题得到根治。
*   **无心之殇与事件雪崩**：找回了因合并代码丢失的实体中枢 `Update()`，让动作得以推进。同时发现了重大的监听防抖缺失 —— 用 `LocalRunner` 对各个 State 中的 `OnComplete` 进行了严管把控与 `OnExit` 的拦截，彻底拔除了每播一次就增加一条监听回调的地雷。

### 4. 闪避充能与管控体系 (Evade System Limit)
*   **配置层暴露**：为 `CharacterConfigSO` 引入了 `evadeLimitedTimes` (连闪上限) 与 `evadeCoolDown` (充能回转间隔) 参数，正式开放给策划进行角色性能与战斗平衡的定制。
*   **实体中枢轮询管控**：将次数与计时器 (`EvadeCount` / `EvadeTimer`) 整合到 `CharacterEntity` 中执行统一管理与 Update 削减计时。在每次动作切实播放触发时调用 `RecordEvade()`，实现精细控制。
*   **预输入拦截闭门羹**：重构了 `CharacterEvadeState` 里的指令接收闭环。当玩家处于连闪窗口期时，预输入的进入不仅看状态机的脸色，更受制于 `CanEvade()` 前置判断。强制阻断了耗尽次数后依赖后摇预输入强行续闪的可能。

### 5. 其他修正与底座固化
*   **统一动作全量预加载**：为了防范实战中初次释放大型 Timeline 以及后续实例化时的瞬间卡顿，在 `CharacterConfigSO` 里补充了 `GetAllActionConfigs()` 提取自身全链路技能数据引用，并在 `ActionManager` 添加 `PreloadCharacterActions` 一次性完成异步管线预热。
*   **启动依赖挂载订正**：纠正了 `GameRoot` 初始化流程向技能运行沙盒单例注入自定义资源加载器时刻的实例拼写谬误（`SkillAssetLoaderAdapter` 改为 `SkillAssetLoader`）。
*   **回调真空期补救流转**：过去在 `CharacterSkillState` 与 `CharacterEvadeState` 触发 `OnSkillEnd` （动作结束回调）时，并没有处理去向操作，导致如果在无后摇派生的情况下动作僵死在原地。通过补偿根据着地与否自动退回 `CharacterGroundState` 或 `CharacterAirborneState` 的判断，成功补上了这条流转裂缝。

## 遗留与计划
*   **评价**：重构非常凶险但结果拔群。不仅战斗手感因为底层优化未受污染，基础移动的手感甚至更加具有潜力，由于全部交托时间轴管控，未来开发策划可以轻而易举地在任意 `Walk` 这类日常行走中配置关键帧震屏甚至添加步态打击特效。
*   **下一步**：在系统全面跑通的前提下，接下来的工作重心将正式回归“手感”本身。如：打击顿帧实现（HitStop / TimeFreeze）、打击特效（Hit Effect）的配置规范、以及结合现有的 `ActionSystem` 对镜头震动产生联动反应。
