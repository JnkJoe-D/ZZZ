## 工作概览
今日的开发工作经历了深刻的指令集与连段系统重构。我们的核心目标是——**统一决策，时间轴自治**。
原来我们的“指令缓冲”与“派生校验”强耦合在各个具体的 FSM 状态内（比如 `CharacterSkillState` 冗长的方法逻辑），这就导致了一旦我们想要给闪避状态也加上连段（比如连闪），就必须复制粘贴一整套繁杂逻辑。
现在，我们抽离出了一个全局战斗大脑 —— `ComboController` 和 `CommandBuffer`。不仅极大程度上削薄了原有状态机的厚度，甚至从根源上摆脱了同一时间点只能允许读取单一 Timeline 窗口切片的隐患。

特别是在测试过程中，挖掘并且扫平了编辑器 Timeline 配置与真实游戏输入系统之间天然的鸿沟。针对连段跳转无法“根据摇杆动态判定向前后闪避”的疑难杂症，通过底层拦截机制进行了智能抹平，为后续负责调优手感的动作设计师免除了巨大的心智负担。

## 详细工作记录

### 1. 指令缓冲与连段判定大一统 (Centralized Input & Combo)
*   **公共池子建立 (`CommandBuffer`)**：为按键添加带时间戳的入队封存机制，精准计算每一个按键在池子中存活的分秒，藉此在连段时可精确分辨这是“预输入指令”还是“此帧现敲的瞬间指令”。
*   **中央处理器 (`ComboController`)**：接盘原属于各独立 State 的派生职权。每次接受玩家动作或窗口交替时，严格拿玩家的合法指令和当前主角激活的所有时间轴轨道做对比。

### 2. 多轨并行与时间优先倒置 (Multi-Track Priority)
*   **循环核芯革命**：最初的判定机制是以“当前存活的窗口”主导循环，导致如果是攻击轨道排在闪避轨道前方，玩家明明最后按的是闪避，反而会抽出上一个延时的攻击预输入错误触发。现已彻底将循环倒置：**外层寻访指令队列（严格按照从老到新时间戳），内层遍历合法窗口**。
*   **开枝散叶的效果**：这看似简单的两行对调，立刻宣告了引擎从此具备支持**真正意义上的同帧多轨道同时并列裁决**的功能。从此你可以在 SkillEditor 中放肆地开立三条甚至四条 `ComboWindowTrack` 分别响应攻击、特技和闪避！

### 3. FSM 状态瘦身大减负 
*   **技能态 (`CharacterSkillState`)**：所有那些试图“缓存攻击”“缓存蓄力”“缓存大招”“手动清查是否有 Buffer”的代码皆作古。它现在唯一要做的，就是纯洁地把每次从 `InputProvider` 听来的触发，原封不动上报给大脑即可。
*   **闪避态 (`CharacterEvadeState`)**：同上。为了对齐最高战斗标准，现在的闪避状态也全盘订阅了所有的攻击与二次闪避事件，彻底融入了全局打断网中。

### 4. 彻底解决前闪/后闪的静态羁绊 (Dynamic Direction Intercept)
*   **痛点**：由于 SkillEditor 上的过渡连线目标往往限定死一份静态 `ActionConfigSO`，策划在绘制“接受闪避打断”的触发区间时如果只填前闪的 Asset，实机测试也只能永远前闪，背离了动作游戏的自然要求。
*   **底层介入**：已在 `ComboController` 的执行匹配断面上加入动态识别：凡是检测到玩家打出的触发签名为 `BufferedInputType.Evade` 时，自动架空 Timeline 上写死的目标参数，改从实体层向 `InputProvider` 获取摇杆朝向，并智能读取绑定的全局 `EvadeFront` 或 `EvadeBack`。这让策划后续连线再也不用在乎目标去向！

### 5. 高级战斗系统问题清扫 (Bug Fixes)
*   **错误重定向下的状态机冷冻**：在重构测试环节期间，发生了角色在连线闪避时冻结在原地的恶性问题。经追踪是由动作类别判定（Category == Evade）因使用者未赋类型值或者 `null` 导致的靶向错误跳转。通过结合修改循环体以及强化 FSM `ChangeState` 处理保底手段，根治了时间停滞 BUG。
*   **陈年指令卡死合法瞬间新键**：过去的队列判定因为在废弃旧获取老按键逻辑后迭代出现残缺，造成旧长按残留项卡死后续 `InstantOnly` 按键匹配。最新架构已支持扫清滞留物，每次成功派生即强制洗牌重置，动作手感由此变得极其干净利落。
