# 指令模型兼容桥说明

## 目标

当前系统还没有一次性切到全新的 `CommandType / CommandPhase / Payload` 路由体系，因此需要一个过渡层：

- 现有逻辑继续兼容 `BufferedInputType`
- `CommandBuffer` 内部开始存更完整的 `CharacterCommand`
- 后续拆分 `EvadeFront / EvadeBack`、拆分长按阶段、补充移动命令时，不需要再重写整条缓冲链

## 当前做法

- 输入处理器仍然把输入交给 `ComboController.OnInput(BufferedInputType)`
- `ComboController` 在入缓冲前，把旧枚举翻译成新的运行时命令对象
- `CharacterCommand` 现在已经包含：
  - `InputType`
  - `CommandType`
  - `CommandPhase`
  - `CommandPayload`
  - `Timestamp`

## 当前价值

- 旧的 `ComboTransition.RequiredCommand` 不需要立刻重配
- 当前的 `ComboWindow + OutTransitions` 逻辑不需要推翻
- 后续如果把 `EvadeFront / EvadeBack` 收敛成统一的 `Evade`，可以直接利用 `Payload.DirectionSnapshot`
- 后续如果加入 `MoveStarted / MoveStopped`，可以直接接入 `CommandType`

## 还没有做的部分

- `ComboTransition` 仍然以 `BufferedInputType` 为触发条件
- `ContextRoutes` 也仍然复用 `ComboTransition`
- `CommandType / CommandPhase` 目前主要作为运行时桥接字段存在，尚未成为主路由键

## 下一步建议

1. 先把 `EvadeFront / EvadeBack` 在运行时解析层收敛成统一 `Evade`
2. 再给 `ComboTransition` 增加对新命令模型的可选支持
3. 最后逐步把核心路由从 `BufferedInputType` 切到 `CommandType + Phase + Payload`
