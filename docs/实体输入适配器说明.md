# 实体输入适配器说明

## 这轮调整

`CharacterEntity` 之前自己承担了两件事：

- 持有角色运行时核心对象
- 手写输入事件的绑定、解绑和逐条转发

现在第二部分已经被抽到了：

- `CharacterInputEventAdapter`

## 当前职责划分

### CharacterEntity

- 持有 `ActionPlayer / CommandBuffer / ComboController / RuntimeData`
- 持有当前状态机
- 在生命周期里让输入适配器绑定和解绑 `IInputProvider`

### CharacterInputEventAdapter

- 订阅 `IInputProvider` 的输入事件
- 根据当前状态读取 `IInputCommandHandler`
- 把输入事件转发给当前 handler

## 当前价值

- `CharacterEntity` 不再需要维护一串私有输入回调函数
- 输入桥接逻辑有了单独落点
- 后续如果要把 handler 改成统一 `Handle(CommandType, CommandPhase, Payload)`，优先改这里即可

## 与前几轮的关系

这一层配合前面已经完成的改造后，输入链路现在更像：

`IInputProvider -> CharacterInputEventAdapter -> IInputCommandHandler -> ComboController -> ActionPlayer`

这已经比最初“Entity 上一堆散落私有回调 + 各状态手搓动作播放”要清晰得多。
