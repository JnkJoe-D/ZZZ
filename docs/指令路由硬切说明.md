# 指令路由硬切说明

## 本次改动

- 删除了旧的 `BufferedInputType -> ComboTransition -> OutTransitions` 主路由链。
- 路由层改为两类：
  - `LocalActionRoute`：挂在动作资产上，负责当前动作自己的局部派生。
  - `ContextRoute`：挂在 `CommandContextConfig` 上，负责当前上下文的默认入口。
- 路由触发键统一为：
  - `CommandType`
  - `CommandPhase`
  - `CommandPayload`
- `EvadeFront / EvadeBack` 不再进入中枢语义层，输入适配后统一上报 `CommandType.Evade`。
- `ActionController` 中的 legacy fallback 已移除，当前上下文动作入口只读 `CommandContextConfig`。

## 现在的主流程

`IInputProvider -> CharacterInputEventAdapter -> CharacterCommand -> CommandBuffer -> ActionController`

中枢处理顺序：

1. 先尝试当前动作的 `LocalRoutes`
2. 再尝试当前上下文的 `ContextRoutes`
3. 命中后由中枢统一设置 `NextActionToCast` 并切状态

## 数据侧新结构

### ActionConfigAsset

- 字段从 `OutTransitions` 改为 `LocalRoutes`
- 适合配置：
  - 普攻连段
  - Dash 冲刺普攻
  - 当前动作特有取消窗口

### CommandContextConfig

- 上下文配置里的 `Routes` 现在是 `ContextRoute`
- 适合配置：
  - `GroundIdle` 普攻起手
  - `GroundJog` 特殊技起手
  - `GroundDash` 默认入口
  - `Backswing` 可接指令

## 当前影响

- 旧的 `OutTransitions` 和旧上下文路由数据不会自动兼容，需要按新结构重配。
- 如果某个上下文没有配置 `ContextRoutes`，那么该上下文不会再走旧的硬编码默认出招。
- `DashAttack` 现在推荐只通过 `Dash` 动作自己的 `ComboWindow + LocalRoutes` 配。

- 调试 HUD 现在会记录最近一次命中的路由来源和路由细节，方便区分是 `LocalRoute` 还是 `ContextRoute`
