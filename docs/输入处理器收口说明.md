# 输入处理器收口说明

## 这轮做了什么

`DefaultInputCommandHandler`、`ComboInputCommandHandler`、`DashInputCommandHandler` 原本都在重复做同一件事：

- 把输入事件转发给 `ComboController`

现在它们已经统一收口到一个公共基类：

- `ForwardingInputCommandHandler`

三个具体 handler 目前只是语义别名，方便状态机继续按原来的名字引用。

## 当前价值

- 去掉了三份完全重复的转发逻辑
- 后续如果某个状态需要特殊改写一条输入映射，只需要覆写对应方法
- 以后如果把输入彻底切到 `CommandType + Phase + Payload`，只需要改这一层的公共转发逻辑

## 现在的定位

这一步还不是终态，它只是把输入 handler 变成一个更干净的适配层。

后续真正的终态会是：

- 输入 handler 只负责把输入适配成统一命令
- 状态机不再关心旧的 `BufferedInputType` 细节
- `ComboController` 最终升级为真正的 `CommandController`
