# 统一闪避命令过渡说明

## 当前状态

系统目前进入了一个兼容过渡态：

- 旧配置仍然可以继续使用 `EvadeFront / EvadeBack`
- 新配置已经可以开始使用通用的 `Evade`
- 运行时会根据输入时的移动意图决定优先解析前闪还是后闪

## 兼容规则

### 1. 旧配置

如果 `ComboTransition.RequiredCommand` 仍然是：

- `EvadeFront`
- `EvadeBack`

那么行为保持不变。

### 2. 新配置

如果 `ComboTransition.RequiredCommand` 配成：

- `Evade`

那么它会同时匹配：

- `EvadeFront`
- `EvadeBack`
- 未来真正统一后的 `Evade`

## 运行时解析

当命中通用 `Evade` 路由时：

- 若输入瞬间存在有效移动输入，则优先解析为前闪
- 若没有有效移动输入，则优先解析为后闪
- 若优先目标未配置，则回退到另一侧闪避动作

## 当前推荐

- 动作本地派生里，如果你已经明确需要区分前后闪，继续用旧配置也没问题
- 上下文默认入口里，推荐开始优先配通用 `Evade`
- 等后续输入层真正收敛到单一闪避命令后，再逐步减少 `EvadeFront / EvadeBack` 的直接使用
