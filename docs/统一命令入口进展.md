# 统一命令入口进展

## 这轮变化

输入链路又往前收了一层：

- `CharacterInputEventAdapter` 现在负责把输入事件直接翻译成 `CharacterCommand`
- `IInputCommandHandler` 已经收敛为统一入口：`Handle(CharacterCommand)`
- `ComboController` 现在可以直接接收 `CharacterCommand`

## 当前链路

`IInputProvider -> CharacterInputEventAdapter -> IInputCommandHandler.Handle(CharacterCommand) -> ComboController -> CommandBuffer`

## 当前意义

- handler 层已经不再暴露一组零散的输入方法名
- 命令对象的构造逻辑不再只藏在 `ComboController` 里
- 后续如果继续把主路由键从 `BufferedInputType` 切到 `CommandType + Phase + Payload`，这一步已经把接口形状准备好了

## 当前约束

- `BasicAttackHoldStart` 目前仍保持旧行为，不直接转发到中枢
- 原因是当前主路由仍以 `BufferedInputType` 为键，若直接转发 `HoldStart`，会和 `Hold` 的既有语义重叠
- 等后续路由正式开始消费 `CommandPhase` 后，再把 `HoldStart` 单独放开会更稳
