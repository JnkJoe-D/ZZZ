# 动作指令中枢当前进展

## 当前已落地

- 非受击状态下的 `ActionPlayer.PlayAction` 已经统一经由 `ComboController` 调用
- `Idle / Jog / Dash / Stop / Skill / Evade` 的播放入口已经收口
- `Dash` 动作仍然支持通过 `ComboWindow + OutTransitions` 配置本地派生
- 新增了角色级 `CommandContextConfig`，用于承载各个 `CommandContextType` 的默认入口

## 新增配置

### 1. CharacterConfigAsset

新增字段：

- `CommandContextConfig`

用途：

- 给当前角色挂一份上下文默认路由配置
- 控制器会优先读取这份配置
- 若未配置，则回退到旧的硬编码默认逻辑

### 2. CommandContextConfig

位置：

- `Assets/GameClient/Config/Action/CommandContextConfig.cs`

结构：

- `ContextRouteGroups`
- 每个 `Group` 对应一个 `CommandContextType`
- 每个 `Group` 下配置若干条 `ComboTransition`

## 配置约定

- `ActionConfigAsset.OutTransitions`
  - 继续用于动作本地派生
  - 例如普攻连段、DashAttack、特殊窗口派生

- `CommandContextConfig`
  - 用于上下文默认入口
  - 例如 `GroundIdle / GroundJog / GroundDash / GroundStop / Backswing`

- `ComboTransition.RequiredWindowTag`
  - 在 `OutTransitions` 中正常生效
  - 在 `CommandContextConfig` 中会被运行时忽略

## 当前推荐迁移顺序

1. 先给角色挂上 `CommandContextConfig`
2. 先迁移 `GroundIdle / GroundJog / GroundStop / Backswing` 的默认入口
3. `GroundDash` 只保留默认兜底，`DashAttack` 继续优先放在 Dash 动作自己的 `ComboWindow + OutTransitions`
4. 确认新配置跑通后，再逐步删掉 controller 里的旧硬编码 fallback

## 当前状态的意义

现在系统已经进入一个比较稳的过渡态：

- 老角色不配新资产不会坏
- 新角色可以开始把默认入口迁到数据
- `Dash` 这种强依赖动作窗口的派生仍然保持在动作本地配置里
- 后续继续往 `Command + Context + Route + Resolver` 体系演进时，不需要再推翻这轮改动
