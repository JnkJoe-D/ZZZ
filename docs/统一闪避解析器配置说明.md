# 统一闪避解析器配置说明

## 目的

`Evade` 在指令层只保留一个统一意图，但实际落地动作仍然可能是：

- 前闪
- 后闪

这两者不再通过两个不同的核心指令来区分，而是通过解析器在运行时根据输入快照自动选择。

## 当前架构

运行时链路如下：

`IInputProvider -> CharacterInputEventAdapter -> CharacterCommand(Evade) -> LocalRoute / ContextRoute -> EvadeCommandActionResolver -> FrontEvade / BackEvade`

含义是：

- 路由层只负责判断“这里能不能闪避”
- 不负责写死“这里一定播前闪”还是“这里只能播后闪”
- 如果路由显式填写了 `NextAction`，则按显式动作播放
- 如果路由把 `NextAction` 留空，则交给 `EvadeCommandActionResolver` 根据输入方向自动分流

## 推荐配置方式

对于“技能/普攻/闪避动作在某个窗口内允许闪避打断”这种需求，推荐这样配：

- `RequiredType = Evade`
- `RequiredPhase = Started`
- `NextAction = 空`

如果是局部动作派生，则配在 `LocalRoutes`。

如果是上下文默认入口，则配在 `ContextRoutes`。

## 什么时候应该显式填写 NextAction

只有在设计上明确要求“这个入口必须固定动作”时，才建议填写 `NextAction`。例如：

- 某个特殊状态只允许后撤闪
- 某个剧情/机关状态只能播固定闪避动作

除此之外，普通战斗态、技能打断态、后摇态，推荐都走“留空 + 解析器自动选变体”。

## 当前解析规则

`EvadeCommandActionResolver` 当前规则：

- 有有效移动输入时，优先前闪
- 无有效移动输入时，优先后闪
- 优先项没配时，自动回退到另一侧

这样可以保证：

- 输入层只保留一个统一 `Evade`
- 配置层不用重复写两条几乎一样的前闪/后闪路由
- 不同角色仍然可以通过各自 `CharacterConfigAsset` 的 `evadeFront / evadeBack` 配出不同表现

## 当前项目里的应用

安比当前这批动作已经切到该模式：

- 上下文路由中的 `Evade`
- 普攻、特殊技、终极技、冲刺普攻等动作上的闪避打断路由

这些路由现在命中后，如果 `NextAction` 为空，就会自动根据输入方向选择前闪或后闪。
